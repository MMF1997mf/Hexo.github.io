在Linux系统中将ES6文件转换为ES5文件并且打包

Share

1.在Linux中安装node

1.下载对应版本的Node.js(https://nodejs.org/en/download/)

2.将下载的代码传输到Linux系统的根目录下的opt文件夹下

对下载的文件进行解压 tar -xvf node-v8.11.3-linux-x64.tar.xz

将解压后的文件移动到你想移动的位置 mv /node-v8.11.3-linux-x64.tar.xz /位置

3.配置环境变量
1）

2)按i进入编辑模式

3）在末尾添加

4.按ESC键退出编辑模式，:wq退出vim

5.重新启动你的系统

6.测试(在任何目录都可以)

7.Node.js安装完成

2.搭建环境并且创建一个项目

1.在该项目下进行初始化

$ npm init

使用-y参数创建默认package.json文件

$ npm init -y

2.使用模块化方式编写代码
例：user.js

3.第三方模块的调用,下载到当前目录，供当前目录使用

3.转换

3.1手动转换

1.babel的安装

$ npm install babel-cli –global

一般会出现权限的错误，因为安装的地址不是全局的(如果没有可以忽略)

$ sudo chown -R $(whoami) $(npm config get prefix)/{lib/node_modules,bin,share}

2.安装bebel-preset-latest预设

$ npm install babel-preset-latest –save-dev

3.配置环境

$ vim .babelrc

进入后按i键进入编辑模式

4.转换(手动将ES6转换为ES5)，查看app01文件夹即可看见转换后的es5文件

$ babel 1-hello.js -o 1-hello-es5.js

3.2自动转换并且打包

1.安装打包器 webpack

$ npm ibstall webpack webpack-cli –save-dev

2.添加配置文件 webpack.config.js

3.为package.json添加脚本

$ vi package.json

4.执行打包(自动将ES6转换为ES5)

$ npm run build

5.打包完成后查看app01文件夹是否有dist文件夹

6.查看dist文件夹下的bundle.js文件，就是我们的劳动成果了

7.用文件传输将bundle.js传输到本地，然后编写html文件，并且将bundle.js文件导入，运行即可看到结果

8.但是bundle.js任然是ES6文件，所以我们还需要将其转化为ES5

1）安装babel-lodaer

$ npm install babel-core babel-loader –save-dev

安装完成后：

2)安装预设

$ npm install babel-preset-latest –save-dev

3)配置文件 .babelrc

$ vi .babelrc

4)在webpack中集成

$ vi webpack.config.js

9.转化打包，但是由于安装babel-lodaer时的版本不匹配，所以我们还需要重新安装一下babel-lodaer

$ npm install babel-loader@7 –save-dev
$ vi package.json

运行完成后进入package.json看到版本变成下图，则表示更改成功

10.打包转换

$ npm run build

11.打包成功后查看dist文件下的bundle.js文件，如果你的箭头函数转化为function函数，则表示转换成功

如何在GitHub上传代码

Share

1.在GitHub注册账号并新建项目

1.登录上注册的GitHub账号后

2.然后进入到GitHub的项目仓库创建页面

3.完成后进入到创建项目的页面，若要进行项目名称的修改或者删除等的修改，都可以点击setting

2.本地git操作

1.安装Git，并且打开Git CMD，cd命令进入到对应的文件夹

2.初始化一个git仓库

$ git init

3.查看文件夹内文件的状态(若有新增的文件，则会变红)。

$ git status

4.若有新增的文件，则把文件交给git追踪

$ git add *

5.再次查看文件的状态

6.对新增的文件进行定义和说明(英文的引号)

$ git commit -m “定义的新增文件的信息”

6.若是第一次使用，则会询问你是谁

7.进行远程连接

$ git remote add origin GitHub项目的地址

8.将远程仓库中的代码更新都本地仓库中

$ git pull origin master –allow-unrelated-histories
–allow-unrelated-histories 以后可以不需要

9.将本地仓库中的文件导入到远程仓库中，完成

$ git push origin master
